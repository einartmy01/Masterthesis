\section{Results}\label{sec:results}

This chapter presents the experimental results obtained from two groups of experiments. The transport protocol tests (T1-T6) and the codec configuration tests (C1-C9). Each group is evaluated across the five metrics system latency, packet loss rate, throughput, CPU usage on the sender, and visual quality. Interpretation and comparison are reserved for Chapter~\ref{sec:discussion}.

Test C4 (H.264 GPU encoding) is missing transit logger data due to a logging error during the experiment. Transit latency for test C4 is therefore missing, but in result of end-to-end latency result from test XX, as it match throughput and should therefore have very similar transit latency, is used to create a comparable result.

\subsubsection{Packet Loss Rate}\label{subsubsec:transport_packetloss}

Packet loss was calculated from RTP sequence numbers by logging ever packet sent and every packet received and matching the loggings and calculate how many packets the sender sent that was not received. For the entire experiment no excess packages was received that was not sent, symbolising a stable receiver logging.

\input{Sections/Tables/Results/T-packetloss.tex}

\subsubsection{Throughput}\label{subsubsec:transport_throughput}

Throughput was logged at the sender side and reflects the volume of encoded video data leaving the pipeline per second. Under unlimited bandwidth conditions (T1--T4), throughput reflects the natural output of the encoder with a bandwidth of "XXX". Under the poor bandwidth condition (T5 and T6), throughput is additionally constrained by the available link capacity. The output measured is included the Tailscale wrapping adding a small percentage to the packets size, this is adjusted for in the constraint tests (T5 and T6) by adding a buffer at the constraint.

\input{Sections/Tables/Results/T-throughput.tex}


\subsubsection{CPU Usage on Sender}\label{subsubsec:transport_cpu}

CPU usage was sampled at regular intervals during each test using the \texttt{sar} utility from the \texttt{sysstat} package. The values reported represent the average user-space CPU load across all cores during the active streaming period. Differences between T1 and T3 isolate the computational overhead of SRTP encryption.

\input{Sections/Tables/Results/T-cpu.tex}

\subsubsection{Visual Quality}\label{subsubsec:transport_quality}

Visual quality was assessed on captured frames using the BRISQUE no-reference image quality metric. Lower BRISQUE scores indicate better perceptual quality. Under poor bandwidth conditions (T5 and T6), compression artefacts and frame losses are expected to increase BRISQUE scores relative to the unlimited bandwidth baseline.

\subsection{Codec Configuration Tests (C1--C9)}
\label{sec:results_codec}

The codec configuration tests examine how codec choice, resolution, and encoding hardware affect streaming performance. All tests used UDP--RTP transport with unlimited bandwidth, isolating codec-related variables from transport effects. Table~\ref{tab:codec_overview} provides an overview of all nine configurations. \subsubsection{Throughput}\label{subsubsec:codec_throughput}

Throughput was logged at the sender side and reflects the volume of encoded video data leaving the pipeline per second. Codec choice had a substantial impact on throughput. MJPEG (C9) produced significantly higher throughput due to intra-frame-only encoding, while H.265 CPU configurations (C5, C7) produced lower throughput driven by the higher compression ratio of HEVC.
